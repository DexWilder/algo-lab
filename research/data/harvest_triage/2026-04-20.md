## 5-gate screen reference

Applied to notes whose parent family is crowded (Tokyo/pre-London,
close-session short, MCL-session, pre-London breakout variants). Note
must pass ≥1 gate or it is pile-on.

- **G1** — new factor exposure
- **G2** — new asset/session logic
- **G3** — blocker-clearing evidence
- **G4** — genuine canonical family split (vs rephrasing existing canonical)
- **G5** — materially different implementation path

Non-crowded-family notes (new_family parent, novel factor/asset) are
accepted without gate analysis — the 5-gate screen is specifically a
crowded-family filter.
